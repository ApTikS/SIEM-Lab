Лог №2: Установка и первичная настройка Wazuh Central Components

Цель: Развертывание системы мониторинга Wazuh (All-in-one) и решение проблемы с аутентификацией в веб-интерфейсе.

Среда: Сервер Ubuntu (целевая машина), Kali Linux (клиент для доступа к Dashboard).
1. Установка Wazuh

Для быстрой установки всех компонентов (Indexer, Server, Dashboard) использовался официальный скрипт автоматического развертывания.

Действие: Выполнение команды в терминале Ubuntu-сервера:
Bash

curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh && sudo bash ./wazuh-install.sh -a

Примечание: флаг -a запускает установку всех компонентов на одном хосте.

2. Проверка доступа и описание проблемы

Для доступа к веб-интерфейсу (Wazuh Dashboard) использовался браузер на удаленной машине (Kali Linux).

    URL: https://<IP_адрес_сервера>:443

    Проблема: При вводе корректных данных, предоставленных установщиком, система отклоняла попытки входа (ошибка аутентификации). Многократная проверка ввода подтвердила неработоспособность стандартного пароля.

3. Решение: Смена пароля администратора

Для восстановления доступа было принято решение принудительно сменить пароль встроенного пользователя admin с помощью внутреннего инструмента Wazuh.

Алгоритм действий:

    Перейти в директорию с инструментами безопасности индексатора:
    cd /usr/share/wazuh-indexer/plugins/opensearch-security/tools/

Запустить скрипт смены пароля:

    sudo bash wazuh-passwords-tool.sh -u admin -p <Ваш_Новый_Пароль>

После успешного выполнения скрипта и завершения процесса обновления хэшей, необходимо перезагрузить сервер для корректного применения изменений и синхронизации всех служб:

    sudo reboot

Результат

После перезагрузки сервера вход в веб-интерфейс Wazuh Dashboard по адресу https://<IP_адрес_сервера> с новым паролем выполнен успешно. Система готова к работе и подключению агентов.
